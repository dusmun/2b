The "Chainsaw Manufacturing" process begins with the "Order received" event in the Administrative lane, signaling the start of the workflow. Following this, the task "Extract chainsaw specifications from order" is performed to gather the necessary details about the order. Once the specifications are extracted, the next step is to "Create production order in ERP," which formalizes the order within the system.

After the production order is created, the process moves to the Procurement lane, where parts are ordered from three different online sources. The tasks "Order parts from online source #1," "Order parts from online source #2," and "Order parts from online source #3" are executed in parallel, ensuring that all necessary components are sourced simultaneously. The process then waits for the events "Parts from source #1 received," "Parts from source #2 received," and "Parts from source #3 received," which indicate that the ordered parts have arrived.

Once all parts are received, the process checks whether "All parts passed manual inspection?" This decision point leads to two possible branches. If the answer is yes, the process continues to the next decision point, "Chainsaw type?" If the answer is no, the process diverts to the task "Order replacement parts," followed by waiting for the event "Replacement parts received," before returning to the inspection check.

At the "Chainsaw type?" decision point, the process branches again based on the type of chainsaw being produced. If the chainsaw is electric, the task "Install electric motor" is performed. If it is a combustion engine, the task "Install combustion engine" is executed instead. Following the installation, the task "Assemble first chainsaw" is carried out, leading to the next step of shipping.

Once the first chainsaw is assembled, the task "Ship first chainsaw to customer" is completed, and the process waits for the event "First chainsaw shipped." After this, the process receives "Customer feedback received," leading to the decision point "Customer approved first chainsaw?" If the customer approves, the process moves to "Ship remaining chainsaws to customer," culminating in the "Order fulfilled" end event. If the customer does not approve, the process takes the alternative path to "Refund customer," which concludes with the "Order canceled" end event.

Meanwhile, after the first chainsaw is shipped, the task "Assemble remaining chainsaws" is performed, which is followed by the event "Remaining chainsaws produced." This task is crucial for completing the order, and once it is done, the process can finalize with the shipping of the remaining chainsaws.

Throughout the process, the sub-process "Manually inspect all parts" is invoked to ensure quality control. This sub-process includes the steps "Perform visual inspection," "Perform functional test," and "Document inspection results," after which the results are sent back to the main flow to determine if all parts passed the manual inspection.

In summary, the "Chainsaw Manufacturing" process is a structured flow that involves receiving orders, procuring parts, assembling chainsaws, and shipping them to customers, while ensuring quality control and addressing customer feedback effectively.
